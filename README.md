# 📊 Analyzing Northwind Database Using PostgreSQL
This repository contains a collection of SQL queries and views for analyzing the classic Northwind sample database using PostgreSQL.
The database can be found [here](https://github.com/pthom/northwind_psql). The picture below gives an overview of the northwind database.

![ER Diagram](ER-Diagram.png)

## 🧭 Overview

The project covers:

- Customer analysis
- Product performance
- Revenue aggregation
- Time-based analysis
- Ranking logic
- Growth detection
- Volatility measurement
- Executive KPI reporting

## Goal 2 – Products on Stock (High-Priced, Active Products)
<details>
<summary><strong>Click to expand</strong></summary>

### 🎯 Goal
- Identify products that are not discontinued and have been sold with a unit price greater than 30.  
- The count indicates how often such products appear in order details.

### 🧾 SQL Query
```sql
select
	prod.product_name,
	count(prod.product_name) as sales_count
from products prod
join order_details od on prod.product_id = od.product_id 
where od.unit_price > 30
  and prod.discontinued = 0
group by prod.product_name
order by sales_count desc;
```

</details>

## Goal 9 – Inactive Customers
<details>
<summary><strong>Click to expand</strong></summary>

### 🎯 Goal
- Identify customers who had a gap of at least 365 days between two consecutive orders.

### 🧾 SQL Query
```sql
select
	sub.contact_name,
	sub.date_diff as inactive_days
from (
	select
		o.order_id,
		cus.contact_name,
		o.order_date,
		o.order_date 
		  - lag(o.order_date) over (
		        partition by cus.contact_name 
		        order by o.order_date
		    ) as date_diff
	from orders o 
	join customers cus on o.customer_id = cus.customer_id
) as sub
where sub.date_diff >= 365
order by inactive_days desc;
```

</details>

## Goal 10 – Customer Ranking
<details>
<summary><strong>Click to expand</strong></summary>

### 🎯 Goal
- Rank customers within each country by their total revenue.  
- Customers with the same revenue share the same rank.

### 🧾 SQL Query
```sql
select
	cus.country,
	cus.contact_name,
	rank() over (
		partition by cus.country 
		order by sum(od.unit_price * od.quantity * (1 - od.discount)) desc
	) as customer_rank
from orders o
join customers cus on o.customer_id = cus.customer_id
join order_details od on o.order_id = od.order_id
group by cus.country, cus.contact_name;
```

</details>

## Goal 11 – Employee Performance
<details>
<summary><strong>Click to expand</strong></summary>

### 🎯 Goal
- Evaluate employee performance based on total revenue,  
  number of handled orders, and average sales per order detail.  
- Includes a global ranking by total revenue.

### 🧾 SQL Query
```sql
select
	rank() over (
		order by sum(od.unit_price * od.quantity * (1 - od.discount)) desc
	) as sales_rank,
	emp.employee_id, 
	emp.first_name,
	emp.last_name,
	count(o.order_id) as orders,
	sum(od.unit_price * od.quantity * (1 - od.discount)) as total_sales,
	avg(od.unit_price * od.quantity * (1 - od.discount)) as avg_sales
from orders o
join order_details od on o.order_id = od.order_id
join employees emp on o.employee_id = emp.employee_id
group by emp.employee_id
order by total_sales desc;
```

</details>

## Goal 12 – Customers with Only On-Time Deliveries
<details>
<summary><strong>Click to expand</strong></summary>

### 🎯 Goal
- Identify customers for whom all orders were delivered on or before the required date.

### 🧾 SQL Query
```sql
select
	cus.contact_name,
	min(o.required_date - o.order_date) as expected_delivery_time,
	min(o.shipped_date - o.order_date) as actual_delivery_time
from orders o
join customers cus on o.customer_id = cus.customer_id
group by cus.contact_name 
having min(o.required_date - o.order_date)
     >= min(o.shipped_date - o.order_date);
```

</details>

## Goal 13 – Products with Strictly Increasing Yearly Sales
<details>
<summary><strong>Click to expand</strong></summary>

### 🎯 Goal
- Identify products whose total yearly sales strictly increase year over year without any decline.

### 🧾 SQL Query
```sql
with t_sales_diff as (
	select
		prod.product_name,
		(
			sum(od.unit_price * od.quantity * (1 - od.discount)) >
			lag(sum(od.unit_price * od.quantity * (1 - od.discount)))
			over (
				partition by prod.product_name
				order by extract(year from o.order_date)
			)
		)::int as sales_diff
	from orders o
	join order_details od on o.order_id = od.order_id
	join products prod on od.product_id = prod.product_id
	group by prod.product_name, extract(year from o.order_date)
)
select
	product_name
from t_sales_diff
group by product_name
having min(sales_diff) <> 0
order by product_name;
```

</details>

## Goal 14 – Countries Where a Product Performs Above Its Average
<details>
<summary><strong>Click to expand</strong></summary>

### 🎯 Goal
For each product, identify countries where its total sales  
are above the product's average sales across all countries.

### 🧾 SQL Query
```sql
with t_total_sales_per_country_product as (
	select 
		prod.product_name,
		cus.country,
		sum(od.unit_price * od.quantity * (1 - od.discount)) as total_sales_per_country
	from orders o
	join order_details od on o.order_id = od.order_id
	join products prod on od.product_id = prod.product_id
	join customers cus on o.customer_id = cus.customer_id
	group by prod.product_name, cus.country
),
t_avg_sales_per_product as (
	select 
		product_name,
		avg(total_sales_per_country) as avg_sales_per_product
	from t_total_sales_per_country_product
	group by product_name
)
select 
	asp.product_name,
	tscp.country,
	tscp.total_sales_per_country,
	asp.avg_sales_per_product
from t_avg_sales_per_product asp
join t_total_sales_per_country_product tscp
  on asp.product_name = tscp.product_name
where tscp.total_sales_per_country >= asp.avg_sales_per_product
order by asp.product_name, tscp.country;
```

</details>

## Goal 15 – Products with Continuous Yearly Growth
<details>
<summary><strong>Click to expand</strong></summary>

### 🎯 Goal
Identify products that:
1) Have sales in consecutive years (no gaps)  
2) Show strictly increasing yearly total sales

### 🧾 SQL Query
```sql
with sales_products_per_year as (
	select
	    prod.product_name,
	    extract(year from o.order_date) as order_year,
	    sum(od.unit_price * od.quantity * (1 - od.discount)) as total_sales
	from orders o
	join order_details od on o.order_id = od.order_id
	join products prod on od.product_id = prod.product_id
	group by prod.product_name, extract(year from o.order_date)
),
t_sales_date_flag as (
	select 
		*,
		coalesce(
			order_year - lag(order_year) 
			over (partition by product_name order by order_year),
			0
		) as year_diff,
		(total_sales >
		 lag(total_sales) 
		 over (partition by product_name order by order_year)
		)::int as sales_increase
	from sales_products_per_year
),
t_filter_products as (
	select 
		product_name
	from t_sales_date_flag
	group by product_name
	having bool_and(
		(year_diff = 0) 
		or (year_diff = 1 and sales_increase = 1)
	)
)
select 
	spy.*
from t_filter_products fp
join sales_products_per_year spy
  on fp.product_name = spy.product_name;
```

</details>

## Goal 16 – Products with Low Yearly Sales Volatility
<details>
<summary><strong>Click to expand</strong></summary>

### 🎯 Goal
- Identify products whose yearly sales variability remains within 10% of their minimum observed variation.  
- Variability is measured using the coefficient of variation.

### 🧾 SQL Query
```sql
with t_stats_sales as (
	select
	    prod.product_name,
	    extract(year from o.order_date) as order_year,
	    sum(od.unit_price * od.quantity * (1 - od.discount)) as total_sales,
	    avg(od.unit_price * od.quantity * (1 - od.discount)) as avg_sales,
	    stddev(od.unit_price * od.quantity * (1 - od.discount)) as std_sales
	from orders o
	join order_details od on o.order_id = od.order_id
	join products prod on od.product_id = prod.product_id
	group by prod.product_name, extract(year from o.order_date)
),
t_stats_sales_var_coeff as (
	select
		*,
		std_sales / avg_sales as var_coeff,
		min(std_sales / avg_sales)
			over (partition by product_name) * 1.1 as max_allowed_var
	from t_stats_sales
),
t_filter_products as (
	select 
		product_name
	from t_stats_sales_var_coeff
	group by product_name
	having bool_and(var_coeff <= max_allowed_var)
)
select 
	sts.*,
	stsvc.var_coeff
from t_filter_products fp
join t_stats_sales sts
  on sts.product_name = fp.product_name
join t_stats_sales_var_coeff stsvc
  on stsvc.product_name = sts.product_name
 and stsvc.order_year = sts.order_year;
```

</details>

## Goal 17 - Executive KPI View – Yearly Business Overview

<details>
<summary><strong>Click to expand</strong></summary>

### 🎯 Goal
Provide a yearly executive-level KPI overview including:
- Total sales
- Number of distinct customers
- Top-selling product
- Top-selling category

### 🧾 SQL Query
```sql
drop view if exists exec_kpi_view;

create or replace view exec_kpi_view as
with t_order_base as (
    select
        extract(year from o.order_date) as order_year,
        o.order_id, 
        od.product_id,
        od.unit_price * od.quantity * (1 - od.discount) as total_sales,
        o.customer_id
    from orders o
    join order_details od on o.order_id = od.order_id
), 
t_total_sales_count_customers as (
    select
        ob.order_year,
        sum(ob.total_sales) as total_sales,
        count(distinct ob.customer_id) as count_customers
    from t_order_base ob
    group by ob.order_year
),
t_top_product as (
    select distinct on (ob.order_year)
        ob.order_year,
        p.product_name as top_product,
        sum(ob.total_sales) as total_sales_product
    from t_order_base ob
    join products p on ob.product_id = p.product_id
    group by ob.order_year, p.product_name
    order by ob.order_year, total_sales_product desc, p.product_name
),
t_top_category as (
    select distinct on (ob.order_year)
        ob.order_year,
        c.category_name as top_category,
        sum(ob.total_sales) as total_sales_category
    from t_order_base ob
    join products p on p.product_id = ob.product_id
    join categories c on p.category_id = c.category_id
    group by ob.order_year, c.category_name
    order by ob.order_year, total_sales_category desc, c.category_name
)
select 
    tscc.order_year,
    tscc.total_sales,
    tscc.count_customers,
    tp.top_product,
    tc.top_category
from t_total_sales_count_customers tscc
join t_top_product tp
    on tscc.order_year = tp.order_year
join t_top_category tc
    on tscc.order_year = tc.order_year
order by tscc.order_year;
```

</details>
